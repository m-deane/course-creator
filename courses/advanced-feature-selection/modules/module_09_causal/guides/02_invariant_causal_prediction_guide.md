# Invariant Causal Prediction

## In Brief

Invariant Causal Prediction (ICP), introduced by Peters, Bühlmann, and Meinshausen (2016), identifies causal features without recovering the full causal graph. The key insight: the conditional distribution $P(Y \mid X_S)$ for the true causal parents $S$ is invariant across environments, while distributions involving non-causal features vary. ICP exploits this invariance to recover a subset of the causal parents via hypothesis testing across multiple environments.

## Key Insight

If you run the same regression $Y \sim X_S$ in multiple environments (time periods, experimental conditions, datasets from different locations) and get the same coefficients and residual distribution every time, then $S$ likely contains only causal features. Spuriously correlated features have environment-specific correlations with $Y$ that change as conditions change — they fail the invariance test.

---

## 1. The Invariance Principle

### Causal Features Have Stable Conditional Distributions

Consider a linear SCM where $Y$ is generated by:

$$Y = \sum_{j \in S^*} \beta_j^* X_j + \varepsilon, \quad \varepsilon \sim \mathcal{N}(0, \sigma^2)$$

where $S^*$ is the true set of causal parents of $Y$. The key property is that across all interventional environments $e \in \mathcal{E}$:

$$P^e(Y \mid X_{S^*}) = P(Y \mid X_{S^*}) \quad \forall e \in \mathcal{E}$$

The conditional distribution of $Y$ given its causal parents is the *same* in every environment.

### Why Non-Causal Features Fail Invariance

A non-causal feature $X_k$ that correlates with $Y$ in environment $e_1$ typically does so via a confounding path or a shared cause. When the environment changes, those confounding paths may strengthen, weaken, or reverse. The conditional regression $Y \mid X_k$ is non-stationary across environments.

Formally, let $H_k \notin S^*$ be a non-causal feature with environment-specific correlation. Then there exists at least one environment $e$ such that:

$$P^e(Y \mid X_{S^* \cup \{k\}}) \neq P(Y \mid X_{S^* \cup \{k\}})$$

This is the signal ICP uses to exclude non-causal features.

---

## 2. Mathematical Formulation

### The ICP Hypothesis

For a candidate set $S \subseteq \{1, \ldots, p\}$, define:

$$H_0(S): \quad P^e(\varepsilon_S) = P^{e'}(\varepsilon_S) \quad \forall e, e' \in \mathcal{E}$$

where $\varepsilon_S = Y - \mathbf{X}_S \boldsymbol{\beta}_S$ are the residuals from regressing $Y$ on $X_S$.

The hypothesis $H_0(S)$ states that the *residual distribution* is the same in all environments. This is equivalent to testing:

1. The regression coefficients $\boldsymbol{\beta}_S$ are the same across environments
2. The residual variance $\sigma^2_S$ is the same across environments

### The ICP Algorithm

Peters et al. (2016) propose the following procedure:

**Step 1:** For each candidate set $S \subseteq \{1, \ldots, p\}$ (or a tractable subset):
- Regress $Y$ on $X_S$ separately in each environment $e$
- Test $H_0(S)$ using a pooled invariance test

**Step 2:** Collect all accepted sets:
$$\mathcal{A} = \{S \subseteq \{1, \ldots, p\} : H_0(S) \text{ not rejected at level } \alpha\}$$

**Step 3:** The ICP estimate of the causal parents is:
$$\hat{S}^{\text{ICP}} = \bigcap_{S \in \mathcal{A}} S$$

The intersection of all accepted sets is a conservative estimate of $S^*$ — it contains only features that appear in every invariant model.

### Why the Intersection?

The true causal set $S^*$ satisfies $H_0(S^*)$ (by the causal Markov condition). Therefore $S^* \in \mathcal{A}$ (with high probability for finite $n$). Any accepted set $S$ contains $S^*$ as a subset (up to finite-sample error) or equals $S^*$. The intersection $\bigcap_{S \in \mathcal{A}} S$ is then contained in $S^*$:

$$\hat{S}^{\text{ICP}} \subseteq S^*$$

**ICP is conservative:** it may miss some causal parents (false negatives) but rarely includes non-causal features (low false positive rate). This makes it ideal for causal feature selection where spurious features are more dangerous than missing features.

## 3. The Invariance Test

### Testing Residual Invariance Across Environments

For a given set $S$, we test whether residuals $\varepsilon_S$ have the same distribution across environments. The standard approach:

**Approach 1: Chow Test (Gaussian, linear)**

Fit a pooled regression and a separate regression per environment. Compare via F-test:

$$F = \frac{(RSS_\text{pooled} - \sum_e RSS_e) / (p \cdot (E-1))}{\sum_e RSS_e / (n - E \cdot (p+1))}$$

Under $H_0$, $F \sim F(p(E-1), n - E(p+1))$.

**Approach 2: Two-stage invariance test (Peters et al.)**

Stage 1: Test coefficient stability across environments (linear hypothesis test).
Stage 2: Test residual variance equality across environments (Levene's test or Bartlett's test).
Reject $H_0(S)$ if either stage rejects.

**Approach 3: Invariance via linear models**

```python
from scipy import stats

def test_invariance(y: np.ndarray,
                    X_S: np.ndarray,
                    env_labels: np.ndarray,
                    alpha: float = 0.05) -> tuple[bool, float]:
    """
    Test whether residuals from Y ~ X_S are invariant across environments.

    Parameters
    ----------
    y : array, shape (n,)
        Target variable.
    X_S : array, shape (n, |S|)
        Candidate feature matrix (subset S).
    env_labels : array, shape (n,)
        Integer environment labels (0, 1, 2, ...).
    alpha : float
        Significance level.

    Returns
    -------
    invariant : bool
        True if H0(S) is not rejected.
    p_value : float
        Combined p-value.
    """
    from sklearn.linear_model import LinearRegression

    envs = np.unique(env_labels)
    residuals_per_env = []

    # Fit separate model per environment
    coefs_per_env = []
    for e in envs:
        mask = env_labels == e
        model = LinearRegression(fit_intercept=True)
        model.fit(X_S[mask], y[mask])
        resid = y[mask] - model.predict(X_S[mask])
        residuals_per_env.append(resid)
        coefs_per_env.append(model.coef_)

    # Stage 1: Test coefficient equality (Wald test)
    coef_matrix = np.array(coefs_per_env)
    coef_std = np.std(coef_matrix, axis=0)
    # Use F-test from pooled fit vs per-environment fit
    n = len(y)
    n_envs = len(envs)
    X_S_with_const = np.column_stack([np.ones(n), X_S])

    # Pooled residuals
    pooled_model = LinearRegression(fit_intercept=True)
    pooled_model.fit(X_S, y)
    resid_pooled = y - pooled_model.predict(X_S)
    rss_pooled = np.sum(resid_pooled ** 2)
    rss_separate = sum(np.sum(r ** 2) for r in residuals_per_env)

    p_s = X_S.shape[1]
    df1 = p_s * (n_envs - 1)
    df2 = n - n_envs * (p_s + 1)
    if df2 <= 0:
        return True, 1.0  # Not enough data to test
    f_stat = ((rss_pooled - rss_separate) / df1) / (rss_separate / df2)
    p_coef = 1 - stats.f.cdf(f_stat, df1, df2)

    # Stage 2: Test residual variance equality (Levene's test)
    p_var = stats.levene(*residuals_per_env).pvalue

    # Combine: reject if either test rejects
    p_combined = min(p_coef, p_var) * 2  # Bonferroni correction for two tests
    p_combined = min(p_combined, 1.0)

    return p_combined > alpha, p_combined
```

## 4. Defining Environments

Environments are the most important design choice in ICP. An environment represents a context where the data-generating process has changed in some way — but crucially, the *causal mechanism* $P(Y \mid \text{Pa}(Y))$ remains unchanged.

### What Counts as a Valid Environment?

Environments must represent actual interventions or distributional shifts:

| Environment type | Example | Valid for ICP? |
|---|---|---|
| Different time periods | Q1 2020 vs Q1 2021 | Yes — if causal mechanism stable |
| Different geographical regions | US vs EU dataset | Yes — causal structure same |
| Experimental conditions | Control vs treatment group | Yes — classic interventional setting |
| Different instruments | Natural experiments, IV | Yes — interventions on covariates |
| Arbitrary data subsets | Random split 1 vs split 2 | No — same distribution, no power |
| Outlier samples | High vs low $Y$ values | No — selection bias, not intervention |

### Practical Environment Construction for Financial/Economic Data

**Time-period environments:** Split the time series into non-overlapping windows:

```python
def create_time_environments(dates: pd.DatetimeIndex,
                             n_envs: int = 4) -> np.ndarray:
    """Split time series into n_envs equal-length environments."""
    n = len(dates)
    env_labels = np.zeros(n, dtype=int)
    boundaries = np.linspace(0, n, n_envs + 1, dtype=int)
    for e in range(n_envs):
        env_labels[boundaries[e]:boundaries[e+1]] = e
    return env_labels
```

**Market regime environments:** Use detected regimes (bull, bear, sideways) as environments. Causal features remain predictive across regimes; momentum features do not.

**Cross-sectional environments:** In panel data, use different firms, countries, or assets as environments.

### Minimum Environment Requirements

ICP requires at least two environments, but power increases with more. Practically:
- 2 environments: detects large invariance violations only
- 3–4 environments: good power for moderate violations
- 5+ environments: best power, can detect subtle non-invariances

## 5. ICP vs Standard Feature Selection

### When They Agree

ICP and standard selection (Lasso, Boruta, SHAP) agree when:
- The data is iid across environments (one environment effectively)
- Causal and predictive features coincide (no confounders)
- Sample size is large enough for ICP to have power

### When They Diverge

**ICP selects fewer features than standard methods:**
- Standard methods keep confounders (which predict $Y$ via spurious paths)
- ICP excludes confounders (they fail invariance tests)
- ICP may miss weak causal features (limited power)

**Example — confounder detection:**

```
Season (S) causes both Ice Cream Sales (IC) and Revenue (Y)
Ad Spend (A) causes Revenue (Y)

Standard Lasso: selects {IC, A}  (IC correlates with Y)
ICP across seasons: selects {A}   (IC fails invariance in different seasons)
```

**ICP selects more features than Lasso when:**
- Lasso is over-regularised (misses weak causal features)
- Multiple features have equal causal contribution (Lasso picks arbitrarily among correlated features)

### Feature-by-Feature Comparison

```python
def icp_vs_standard_comparison(icp_set, lasso_set, boruta_set, shap_set,
                               feature_names):
    """Compare ICP features against standard selection methods."""
    sets = {
        'ICP': set(icp_set),
        'Lasso': set(lasso_set),
        'Boruta': set(boruta_set),
        'SHAP': set(shap_set),
    }
    # Features in ICP but not in any standard method: likely confounders excluded by standard
    icp_only = sets['ICP'] - (sets['Lasso'] | sets['Boruta'] | sets['SHAP'])
    # Features in standard but not ICP: likely spurious correlates
    standard_only = (sets['Lasso'] | sets['Boruta'] | sets['SHAP']) - sets['ICP']
    # Consensus: in both ICP and at least two standard methods
    consensus = sets['ICP'] & (sets['Lasso'] | sets['Boruta'] | sets['SHAP'])
    return {
        'icp_only': icp_only,
        'standard_only': standard_only,
        'consensus': consensus,
    }
```

## 6. Limitations of ICP

### Faithfulness Requirement

ICP assumes that if a feature set $S$ is not a superset of $S^*$, there exists at least one environment where the invariance fails detectably. This is a weaker assumption than standard faithfulness but can still be violated.

**Example of violation:** If a confounder affects $Y$ through $X_k$, but the confounder's effect is exactly constant across all available environments, ICP cannot detect the non-causality of $X_k$.

### Finite Environment Power

With only a few environments, ICP has limited statistical power. A feature that is causally irrelevant but happens to have similar regression coefficients across environments will be incorrectly accepted.

**Mitigation:** Use more environments, increase sample size per environment, or lower the significance level $\alpha$.

### Computational Complexity

Exhaustive ICP tests all $2^p$ subsets. For $p > 20$, this is intractable.

**Practical solutions:**
- **Sparse ICP:** Only test subsets up to size $k$ (e.g., $k=5$)
- **Greedy ICP:** Forward selection using invariance test
- **Lasso-ICP:** Use Lasso to pre-screen, then apply ICP to survivors

```python
def greedy_icp(y, X, env_labels, alpha=0.05, max_features=10):
    """
    Greedy ICP: forward selection using invariance tests.

    At each step, add the feature that maintains invariance
    and most reduces pooled residual variance.
    """
    p = X.shape[1]
    selected = []
    remaining = list(range(p))

    while remaining and len(selected) < max_features:
        best_feature = None
        best_rss = np.inf
        for j in remaining:
            candidate = selected + [j]
            X_cand = X[:, candidate]
            is_invariant, p_val = test_invariance(y, X_cand, env_labels, alpha)
            if is_invariant:
                # Compute pooled RSS for ranking
                from sklearn.linear_model import LinearRegression
                model = LinearRegression().fit(X_cand, y)
                rss = np.sum((y - model.predict(X_cand)) ** 2)
                if rss < best_rss:
                    best_rss = rss
                    best_feature = j
        if best_feature is None:
            break  # No more invariant features
        selected.append(best_feature)
        remaining.remove(best_feature)

    return selected
```

### Linear ICP vs Nonlinear Reality

The standard ICP formulation assumes linear structural equations. In practice, nonlinear relationships require extensions.

## 7. Nonlinear ICP Extensions

### Invariant Risk Minimisation (IRM)

Arjovsky et al. (2019) extend the invariance principle to nonlinear settings via **Invariant Risk Minimisation**:

$$\min_{\Phi, w} \sum_e \mathcal{R}^e(w \circ \Phi) + \lambda \left\| \nabla_w \mathcal{R}^e(w \circ \Phi) \Big|_{w=1.0} \right\|^2$$

where $\Phi$ is a nonlinear feature extractor and the penalty enforces that the optimal classifier on top of $\Phi$ is the same across environments.

IRM recovers causal features as the components of $\Phi$ that are used identically across environments.

### Conditional Invariance Test (nonlinear)

Replace the linear residual test with a kernel-based conditional independence test:

$$H_0(S): Y \perp \!\!\! \perp E \mid X_S$$

where $E$ is the environment indicator. This tests whether knowing the environment adds information about $Y$ beyond what $X_S$ already provides.

```python
from sklearn.kernel_approximation import RBFSampler
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import cross_val_score

def nonlinear_invariance_test(y, X_S, env_labels, alpha=0.05):
    """
    Nonlinear invariance test: Y ⊥ E | X_S.
    Tests whether environment predicts Y beyond X_S.
    """
    # Feature for testing: predict environment from X_S and Y
    # If environment is predictable beyond X_S alone, invariance fails
    env_binary = (env_labels > env_labels.mean()).astype(int)  # binary for simplicity
    X_aug = np.column_stack([X_S, y.reshape(-1, 1)])
    X_only = X_S

    # Compare: predict E from X_S alone vs X_S + Y
    clf = LogisticRegression(C=1.0, max_iter=1000)
    score_with_y = cross_val_score(clf, X_aug, env_binary, cv=5).mean()
    score_without_y = cross_val_score(clf, X_only, env_binary, cv=5).mean()

    # If Y adds predictive power for E beyond X_S, invariance violated
    gain = score_with_y - score_without_y
    # Use permutation test for significance
    n_perms = 100
    gains_perm = []
    for _ in range(n_perms):
        y_perm = np.random.permutation(y)
        X_aug_perm = np.column_stack([X_S, y_perm.reshape(-1, 1)])
        score_perm = cross_val_score(clf, X_aug_perm, env_binary, cv=5).mean()
        gains_perm.append(score_perm - score_without_y)

    p_value = np.mean(np.array(gains_perm) >= gain)
    return p_value > alpha, p_value
```

## Common Pitfalls

- **Environments too similar:** If environments represent the same distribution, all features pass the invariance test trivially. ICP has no power.
- **Environments defined by outcome:** Never split environments using $Y$ values (e.g., "high revenue" vs "low revenue"). This introduces selection bias that invalidates the test.
- **Too few environments:** With only 2 environments and $p$ features, the intersection is typically the full feature set. Need at least 3 diverse environments for meaningful selection.
- **Ignoring power:** ICP's false negative rate can be high. Always supplement with standard feature selection and take the union for completeness.
- **Linear-only assumption:** Real relationships are rarely linear. Nonlinear invariance tests (kernel-based or IRM) are preferable when linearity is suspect.
