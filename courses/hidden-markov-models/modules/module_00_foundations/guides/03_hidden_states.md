# Hidden States: Why "Hidden" in Hidden Markov Models?

## In Brief

Hidden Markov Models assume an underlying (hidden) state process that we cannot directly observe. We only see emissions or observations generated by these hidden states. The goal is to infer the hidden states from the observations.

> 💡 **Key Insight:** Financial markets have regimes (bull/bear, high/low volatility) that we cannot directly observe—we only see prices and returns. HMMs model this disconnect between what truly drives the system (hidden states) and what we measure (observations).

## Formal Definition

### Observable vs Hidden Markov Chain

**Observable Markov Chain:**
- State sequence: $q_1, q_2, ..., q_T$ (directly observed)
- We see: States themselves
- Question: What happens next?

**Hidden Markov Model:**
- State sequence: $q_1, q_2, ..., q_T$ (hidden/latent)
- Observation sequence: $o_1, o_2, ..., o_T$ (observed)
- We see: Observations, not states
- Question: What hidden states generated these observations?

### The HMM Structure

```
HIDDEN LAYER:    q₁ → q₂ → q₃ → ... → qₜ  (Markov chain)
                 ↓    ↓    ↓         ↓
OBSERVED LAYER:  o₁   o₂   o₃   ...  oₜ  (emissions)
```

**Key Properties:**
1. **Hidden states** follow a Markov chain: $P(q_t | q_{t-1}, q_{t-2}, ..., q_1) = P(q_t | q_{t-1})$
2. **Observations** depend only on current hidden state: $P(o_t | q_t, q_{t-1}, ..., o_{t-1}, ...) = P(o_t | q_t)$
3. **Conditional independence**: Given the state $q_t$, observation $o_t$ is independent of all past states and observations

### Mathematical Specification

An HMM $\lambda$ is defined by:

$$\lambda = (\Pi, A, B)$$

Where:
- $\Pi = [\pi_i]$ : Initial state distribution, $\pi_i = P(q_1 = s_i)$
- $A = [a_{ij}]$ : Transition matrix, $a_{ij} = P(q_{t+1} = s_j | q_t = s_i)$
- $B = [b_i(o)]$ : Emission distribution, $b_i(o) = P(o_t = o | q_t = s_i)$

## Intuitive Explanation

### The Weather Analogy

Imagine you're locked in a room with no windows. You can't see the weather (hidden state), but you can observe what your friend is wearing when they visit:

```
Hidden States (Weather):
  Sunny → Rainy → Rainy → Sunny

Observations (Clothing):
  Shorts → Umbrella → Raincoat → T-shirt
```

**The Challenge:** Given the clothing sequence, infer what the weather was.

**Why Hidden?** You can't directly measure the weather—you infer it from indirect observations.

### Financial Markets Example

**Hidden States:** Market regimes
- Bull market (trending up)
- Bear market (trending down)
- Consolidation (sideways)

**Observations:** Daily returns
- +2.5%
- -1.3%
- +0.8%
- ...

**Why Hidden?**
- No single number says "we're in a bull market"
- Regime is a latent concept we infer from prices
- Multiple assets might be in different regimes simultaneously
- Transitions between regimes are not instantaneous or obvious

### The Inference Problem

```python
# What we have:
observations = [+0.02, +0.03, -0.01, +0.02, ...]  # Returns

# What we want to know:
states = [Bull, Bull, Bear, Bull, ...]            # Hidden regimes

# How we get there:
# 1. Model: Assume regimes exist and follow Markov dynamics
# 2. Parameters: Learn P(regime transitions) and P(return | regime)
# 3. Inference: Use observations to infer most likely state sequence
```

## Why Use Hidden States?

### 1. Dimension Reduction

**Complexity of observations:**
- Stock returns can take infinite values (continuous)
- Joint distribution of 1000-day returns is intractable

**Simplification with hidden states:**
- Finite number of discrete states (e.g., 2-5)
- State transitions are simple (Markov property)
- Observations become simpler given states

### 2. Interpretability

**Raw observations:**
- Daily return = +1.2%
- Hard to interpret in isolation

**With hidden states:**
- State = "Bull Market" (interpretable concept)
- Returns in bull state: mean +5% annually, volatility 12%
- Actionable insights for decision-making

### 3. Prediction

**Without states:**
- Predict next return from history
- Complex, non-stationary

**With states:**
- Predict next state (simple Markov chain)
- Predict next return given state
- Two simpler problems instead of one complex problem

### 4. Missing Data Handling

HMMs gracefully handle missing observations:

```
States:       q₁ → q₂ → q₃ → q₄ → q₅
Observations: o₁   o₂   ?    o₄   o₅
```

The hidden state layer provides continuity even when observations are missing.

## Code Implementation

### Observable vs Hidden Markov Chain

```python
import numpy as np
import matplotlib.pyplot as plt

class ObservableMarkovChain:
    """Traditional Markov chain - we observe states directly."""

    def __init__(self, transition_matrix, state_names):
        self.A = np.array(transition_matrix)
        self.states = state_names
        self.n_states = len(state_names)

    def simulate(self, n_steps, initial_state=0):
        """Simulate state sequence - all states are observed."""
        states = [initial_state]

        for _ in range(n_steps - 1):
            current = states[-1]
            next_state = np.random.choice(self.n_states, p=self.A[current])
            states.append(next_state)

        return states, states  # observations = states (fully observable)


class HiddenMarkovModel:
    """HMM - we observe emissions, not states."""

    def __init__(
        self,
        transition_matrix,
        emission_distributions,
        state_names
    ):
        self.A = np.array(transition_matrix)
        self.emissions = emission_distributions  # Dict mapping state → distribution
        self.states = state_names
        self.n_states = len(state_names)

    def simulate(self, n_steps, initial_state=0):
        """
        Simulate HMM sequence.

        Returns:
            states: Hidden state sequence (not observed in real scenario)
            observations: Observed emissions
        """
        states = [initial_state]
        observations = [self.emissions[initial_state].sample()]

        for _ in range(n_steps - 1):
            # Transition to next state (hidden)
            current = states[-1]
            next_state = np.random.choice(self.n_states, p=self.A[current])
            states.append(next_state)

            # Generate observation from emission distribution
            observation = self.emissions[next_state].sample()
            observations.append(observation)

        return states, observations


# Example: Market regimes
class GaussianEmission:
    """Gaussian emission distribution."""

    def __init__(self, mean, std):
        self.mean = mean
        self.std = std

    def sample(self):
        return np.random.normal(self.mean, self.std)

    def pdf(self, x):
        return (1 / (self.std * np.sqrt(2*np.pi))) * \
               np.exp(-0.5 * ((x - self.mean) / self.std)**2)


# Define market regime HMM
transition_matrix = [
    [0.95, 0.05],  # Bull → {Bull, Bear}
    [0.10, 0.90]   # Bear → {Bull, Bear}
]

emissions = {
    0: GaussianEmission(mean=0.001, std=0.01),   # Bull: +0.1% daily, 1% vol
    1: GaussianEmission(mean=-0.001, std=0.02)   # Bear: -0.1% daily, 2% vol
}

state_names = ['Bull', 'Bear']

# Create HMM
hmm = HiddenMarkovModel(transition_matrix, emissions, state_names)

# Simulate
np.random.seed(42)
hidden_states, observations = hmm.simulate(200, initial_state=0)

# Visualize the hidden vs observed
fig, axes = plt.subplots(2, 1, figsize=(14, 8), sharex=True)

# Plot 1: Hidden states (what we want to know)
ax1 = axes[0]
ax1.fill_between(range(200), 0, hidden_states, alpha=0.3, step='mid')
ax1.step(range(200), hidden_states, where='mid', linewidth=2, color='darkblue')
ax1.set_ylabel('Hidden State')
ax1.set_yticks([0, 1])
ax1.set_yticklabels(['Bull', 'Bear'])
ax1.set_title('Hidden Layer: True Market Regimes (Unknown in Reality)')
ax1.grid(True, alpha=0.3)

# Plot 2: Observations (what we actually see)
ax2 = axes[1]
ax2.plot(observations, linewidth=1, alpha=0.7, color='darkgreen')
ax2.axhline(0, color='black', linestyle='--', alpha=0.3)
ax2.set_xlabel('Time')
ax2.set_ylabel('Return')
ax2.set_title('Observed Layer: Daily Returns (What We Actually See)')
ax2.grid(True, alpha=0.3)

plt.tight_layout()
plt.show()

print("HMM Simulation Results:")
print(f"Total steps: {len(observations)}")
print(f"Observations range: [{min(observations):.4f}, {max(observations):.4f}]")
print(f"True state changes: {sum(1 for i in range(1, len(hidden_states)) if hidden_states[i] != hidden_states[i-1])}")
```

### Why States are "Hidden"

```python
def demonstrate_hidden_ambiguity(hmm, observations):
    """
    Show why states are hidden: same observation can come from different states.
    """
    # Take an observation near zero
    obs_value = 0.0

    # Compute likelihood under each state
    for state_idx, state_name in enumerate(hmm.states):
        emission_dist = hmm.emissions[state_idx]
        likelihood = emission_dist.pdf(obs_value)
        print(f"P(return={obs_value} | state={state_name}) = {likelihood:.4f}")

    print("\n➜ Same observation has non-zero probability under multiple states!")
    print("➜ We can't deterministically know the state from observation alone.")
    print("➜ Need to use sequence of observations + model parameters.")


demonstrate_hidden_ambiguity(hmm, observations)
```

Output:
```
P(return=0.0 | state=Bull) = 39.8942
P(return=0.0 | state=Bear) = 19.9471

➜ Same observation has non-zero probability under multiple states!
➜ We can't deterministically know the state from observation alone.
➜ Need to use sequence of observations + model parameters.
```

### The Three Fundamental Questions

```python
class HMMQuestions:
    """
    The three canonical problems for HMMs.
    """

    def __init__(self, hmm):
        self.hmm = hmm

    def evaluation(self, observations):
        """
        Problem 1: EVALUATION
        Given model λ and observations O, compute P(O|λ)

        Question: How likely is this observation sequence under our model?
        Use case: Model comparison, anomaly detection
        Algorithm: Forward algorithm
        """
        print("EVALUATION: How likely are these observations under our model?")
        print(f"Observations: {observations[:5]}...")
        print("→ Need to compute P(O|λ)")
        print("→ Algorithm: Forward algorithm (Module 2)")

    def decoding(self, observations):
        """
        Problem 2: DECODING
        Given model λ and observations O, find most likely state sequence Q*

        Question: What hidden states generated these observations?
        Use case: Regime identification, sequence labeling
        Algorithm: Viterbi algorithm
        """
        print("\nDECODING: What hidden states generated these observations?")
        print(f"Observations: {observations[:5]}...")
        print("→ Need to find Q* = argmax_Q P(Q|O,λ)")
        print("→ Algorithm: Viterbi algorithm (Module 2)")

    def learning(self, observations):
        """
        Problem 3: LEARNING
        Given observations O, find model λ* that maximizes P(O|λ)

        Question: What are the best parameters for this HMM?
        Use case: Parameter estimation from data
        Algorithm: Baum-Welch (EM)
        """
        print("\nLEARNING: What parameters best explain these observations?")
        print(f"Observations: {observations[:5]}...")
        print("→ Need to find λ* = argmax_λ P(O|λ)")
        print("→ Algorithm: Baum-Welch (EM) (Module 2)")


problems = HMMQuestions(hmm)
problems.evaluation(observations)
problems.decoding(observations)
problems.learning(observations)
```

## Common Pitfalls

1. **Assuming states are observable**
   - If you can directly see the states, you don't need an HMM
   - HMMs are for latent (hidden) state processes
   - Example mistake: Using HMM when you already have regime labels

2. **Too many states**
   - More states ≠ better model
   - Leads to overfitting and poor generalization
   - Start with 2-3 states, only add more with strong justification
   - Use information criteria (BIC/AIC) for selection

3. **Ignoring the observation model**
   - States alone are not the goal—they're a means to model observations
   - Choose emission distributions carefully (Gaussian, mixture, etc.)
   - Emission model matters as much as transition model

4. **Expecting deterministic states**
   - HMM inference gives probabilities, not certainties
   - "Most likely state" ≠ "definitely in this state"
   - Always consider uncertainty in state estimates

5. **Confusing filtering vs smoothing**
   - **Filtering**: P(state at time t | observations up to time t)
   - **Smoothing**: P(state at time t | all observations)
   - Smoothing is more accurate but requires the full sequence

## Connections

### Builds on:
- **Markov chains**: States follow Markov dynamics
- **Conditional probability**: P(observation | state)
- **Bayes' theorem**: Infer P(state | observation)

### Leads to:
- **Forward-Backward algorithm**: Compute state probabilities
- **Viterbi algorithm**: Find most likely state sequence
- **Baum-Welch algorithm**: Learn parameters from data
- **Kalman filters**: HMMs with continuous states
- **Particle filters**: Non-parametric state estimation

### Related to:
- **Mixture models**: Special case where states don't transition
- **State-space models**: General framework (HMMs are discrete version)
- **Graphical models**: HMMs are temporal Bayesian networks
- **Regime-switching models**: Economic/financial application of HMMs

## Practice Problems

1. **Conceptual Understanding**

   You have a 2-state HMM for weather (Sunny/Rainy) with observations being whether people carry umbrellas (Yes/No). Explain why the weather is "hidden" even though you could just look outside.

   <details>
   <summary>Answer</summary>

   The "hidden" nature is about the mathematical model, not physical reality. In the model:
   - States (Sunny/Rainy) follow Markov dynamics
   - Observations (Umbrella: Yes/No) are generated probabilistically from states
   - The challenge is inferring states from observations only

   Even though we could observe weather directly in reality, the HMM framework treats it as latent to model the probabilistic relationship between weather and human behavior. This is useful when:
   - You only have umbrella data historically
   - You want to predict behavior (umbrella) from weather forecasts
   - The "state" represents something less concrete (e.g., "tendency to prepare for rain")
   </details>

2. **Market Regimes**

   Why is it appropriate to model market regimes as hidden states rather than directly labeling them from price data?

   <details>
   <summary>Answer</summary>

   Reasons regimes are "hidden":
   - No consensus definition of what constitutes a bull vs bear market
   - Transitions are gradual, not instantaneous
   - Different assets may be in different regimes simultaneously
   - Regime is a latent concept we infer from multiple indicators
   - Historical labels would be subjective and look-ahead biased

   HMMs let the data reveal regime structure without arbitrary cutoffs.
   </details>

3. **Observation Ambiguity**

   An HMM has two states (Low Vol, High Vol) with Gaussian emissions:
   - Low Vol: N(μ=0, σ=0.01)
   - High Vol: N(μ=0, σ=0.03)

   You observe a return of 0.02. Can you definitively say which state generated it?

   <details>
   <summary>Answer</summary>

   No—both states can generate this observation:
   - P(0.02 | Low Vol) = pdf of N(0, 0.01) at 0.02
   - P(0.02 | High Vol) = pdf of N(0, 0.03) at 0.02

   Both are non-zero, though Low Vol is less likely (0.02 is 2 standard deviations out).

   To infer the state, we need:
   - Prior state probabilities
   - Transition dynamics
   - Sequence context (what came before/after)

   This is exactly why we need HMM algorithms (Viterbi, Forward-Backward).
   </details>

4. **Design Choice**

   You're modeling customer behavior: {Browsing, Considering, Purchasing}. Observations are webpage visits. Should you use an observable Markov chain or an HMM?

   <details>
   <summary>Answer</summary>

   **Use HMM** because:
   - Customer intent (Browsing/Considering/Purchasing) is latent—we don't directly observe it
   - Webpage visits are imperfect indicators of intent
   - Same webpage might be visited in different states (e.g., product page while browsing vs considering)
   - We want to infer hidden intent from observable behavior

   **Observable chain would be appropriate if:**
   - We only care about page-to-page transitions
   - Page type perfectly indicates state
   - No latent variable involved
   </details>

**Key Takeaway:** States are "hidden" because we observe only indirect evidence (emissions) of an underlying process. HMMs provide a principled probabilistic framework for inferring these hidden states from observations, enabling us to model complex systems where the true state is latent or unobservable.
