# HMM Framework and Definition

## From Markov Chains to Hidden Markov Models

In a standard Markov chain, we observe the states directly. In an HMM, states are **hidden** and we only observe **emissions** generated by those states.

```
Standard Markov Chain:
    State₁ → State₂ → State₃ → ...
    (observed)

Hidden Markov Model:
    Hidden:    State₁ → State₂ → State₃ → ...
                  ↓        ↓        ↓
    Observed:   Obs₁     Obs₂     Obs₃     ...
```

## Formal Definition

An HMM $\lambda = (A, B, \pi)$ consists of:

### 1. Hidden States
$S = \{s_1, s_2, ..., s_K\}$

### 2. Observations
$O = \{o_1, o_2, ..., o_T\}$ (discrete or continuous)

### 3. Initial Distribution
$\pi_i = P(q_1 = s_i)$

### 4. Transition Matrix
$a_{ij} = P(q_{t+1} = s_j | q_t = s_i)$

### 5. Emission Probabilities
$b_i(o) = P(o_t = o | q_t = s_i)$

## The Three Fundamental Problems

| Problem | Question | Algorithm | Use Case |
|---------|----------|-----------|----------|
| **Evaluation** | $P(O \| \lambda)$? | Forward | Model comparison |
| **Decoding** | Most likely state sequence? | Viterbi | Regime identification |
| **Learning** | Best parameters? | Baum-Welch | Model training |

## Implementation

### HMM Class Structure

```python
import numpy as np
from typing import List, Optional, Tuple
from dataclasses import dataclass

@dataclass
class HMMParams:
    """HMM parameter container."""
    pi: np.ndarray      # Initial distribution (K,)
    A: np.ndarray       # Transition matrix (K, K)
    B: np.ndarray       # Emission matrix (K, M) for discrete

    @property
    def n_states(self) -> int:
        return len(self.pi)

class DiscreteHMM:
    """Hidden Markov Model with discrete emissions."""

    def __init__(
        self,
        n_states: int,
        n_symbols: int,
        params: Optional[HMMParams] = None
    ):
        self.n_states = n_states
        self.n_symbols = n_symbols

        if params:
            self.pi = params.pi
            self.A = params.A
            self.B = params.B
        else:
            self._initialize_random()

    def _initialize_random(self):
        """Random initialization."""
        # Initial distribution
        self.pi = np.random.dirichlet(np.ones(self.n_states))

        # Transition matrix
        self.A = np.random.dirichlet(np.ones(self.n_states), size=self.n_states)

        # Emission matrix
        self.B = np.random.dirichlet(np.ones(self.n_symbols), size=self.n_states)

    def generate(self, length: int) -> Tuple[List[int], List[int]]:
        """Generate a sequence of states and observations."""
        states = []
        observations = []

        # Initial state
        state = np.random.choice(self.n_states, p=self.pi)
        states.append(state)

        # Initial observation
        obs = np.random.choice(self.n_symbols, p=self.B[state])
        observations.append(obs)

        # Generate sequence
        for _ in range(length - 1):
            # Transition
            state = np.random.choice(self.n_states, p=self.A[state])
            states.append(state)

            # Emission
            obs = np.random.choice(self.n_symbols, p=self.B[state])
            observations.append(obs)

        return states, observations
```

### Example: Weather HMM

```python
# Hidden states: Sunny (0), Rainy (1)
# Observations: Walk (0), Shop (1), Clean (2)

# Transition probabilities
A = np.array([
    [0.7, 0.3],  # Sunny → Sunny: 70%, Sunny → Rainy: 30%
    [0.4, 0.6]   # Rainy → Sunny: 40%, Rainy → Rainy: 60%
])

# Emission probabilities
B = np.array([
    [0.6, 0.3, 0.1],  # Sunny: Walk 60%, Shop 30%, Clean 10%
    [0.1, 0.4, 0.5]   # Rainy: Walk 10%, Shop 40%, Clean 50%
])

# Initial distribution
pi = np.array([0.6, 0.4])

params = HMMParams(pi=pi, A=A, B=B)
hmm = DiscreteHMM(n_states=2, n_symbols=3, params=params)

# Generate sequence
states, obs = hmm.generate(20)
print(f"Hidden states: {states}")
print(f"Observations: {obs}")
```

## Probability Calculations

### Joint Probability of States and Observations

For a state sequence $Q = q_1, q_2, ..., q_T$ and observations $O = o_1, o_2, ..., o_T$:

$$P(O, Q | \lambda) = \pi_{q_1} \cdot b_{q_1}(o_1) \cdot \prod_{t=2}^{T} a_{q_{t-1}, q_t} \cdot b_{q_t}(o_t)$$

```python
def joint_probability(
    hmm: DiscreteHMM,
    states: List[int],
    observations: List[int]
) -> float:
    """Compute joint probability P(O, Q | λ)."""
    T = len(observations)

    # Initial probability
    prob = hmm.pi[states[0]] * hmm.B[states[0], observations[0]]

    # Subsequent terms
    for t in range(1, T):
        prob *= hmm.A[states[t-1], states[t]]
        prob *= hmm.B[states[t], observations[t]]

    return prob

# Example
jp = joint_probability(hmm, states, obs)
print(f"Joint probability: {jp:.2e}")
```

### Log Probability (Numerical Stability)

```python
def log_joint_probability(
    hmm: DiscreteHMM,
    states: List[int],
    observations: List[int]
) -> float:
    """Compute log joint probability for numerical stability."""
    T = len(observations)

    # Initial
    log_prob = np.log(hmm.pi[states[0]]) + np.log(hmm.B[states[0], observations[0]])

    # Subsequent
    for t in range(1, T):
        log_prob += np.log(hmm.A[states[t-1], states[t]])
        log_prob += np.log(hmm.B[states[t], observations[t]])

    return log_prob
```

## Likelihood of Observations

The total probability of observations (marginalizing over all state sequences):

$$P(O | \lambda) = \sum_{\text{all } Q} P(O, Q | \lambda)$$

**Problem**: For $T$ observations and $K$ states, there are $K^T$ possible sequences!

**Solution**: Dynamic programming (Forward algorithm) - covered in next module.

## Independence Assumptions

HMMs make two key independence assumptions:

### 1. First-Order Markov Property
$$P(q_t | q_{t-1}, ..., q_1) = P(q_t | q_{t-1})$$

### 2. Output Independence
$$P(o_t | q_T, ..., q_1, o_{t-1}, ..., o_1) = P(o_t | q_t)$$

The observation depends only on the current hidden state.

## Common HMM Variants

| Variant | Emissions | Use Case |
|---------|-----------|----------|
| **Discrete HMM** | Categorical | Text, DNA sequences |
| **Gaussian HMM** | Normal distribution | Returns, prices |
| **GMM-HMM** | Gaussian mixture | Speech recognition |
| **Autoregressive HMM** | AR process | Correlated time series |

## Key Takeaways

1. **HMMs extend Markov chains** by adding an emission layer

2. **Three core problems**: Evaluation, Decoding, Learning

3. **Independence assumptions** make inference tractable

4. **Log probabilities** are essential for numerical stability

5. **Multiple variants** exist for different emission types
