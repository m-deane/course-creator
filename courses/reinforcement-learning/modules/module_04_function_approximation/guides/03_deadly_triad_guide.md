# The Deadly Triad

> **Reading time:** ~17 min | **Module:** 4 — Function Approximation | **Prerequisites:** Module 3

## In Brief

The deadly triad refers to the three conditions whose simultaneous combination can cause reinforcement learning algorithms to diverge: function approximation, bootstrapping, and off-policy training. Remove any one condition and convergence is recoverable. Combine all three, and even simple linear methods can produce unbounded value estimates. Understanding the triad explains every major stabilization technique in modern deep RL.

## Key Insight

The triad is not a limitation of specific algorithms — it is a fundamental property of the problem structure. Sutton & Barto Chapter 11.3 proves this via Baird's counterexample: a 7-state MDP where a linear approximator trained with semi-gradient Q-learning diverges to infinity despite a perfect representation and a simple problem.

## 1. The Three Conditions

### Condition 1: Function Approximation

<div class="callout-key">

<strong>Key Point:</strong> ### Condition 1: Function Approximation

Any parameterized value function $\hat{v}(s, \mathbf{w})$ or $\hat{q}(s, a, \mathbf{w})$ where $|\mathbf{w}| \ll |\mathcal{S}|$.

</div>


Any parameterized value function $\hat{v}(s, \mathbf{w})$ or $\hat{q}(s, a, \mathbf{w})$ where $|\mathbf{w}| \ll |\mathcal{S}|$. This includes linear methods, tile coding, RBFs, and neural networks. Tabular methods are exempt — each state has its own parameter, so there is no cross-state interference from weight sharing.

**Why FA matters:** Weight sharing means updating the value estimate for state $s$ changes the estimates for all states with similar features. Under on-policy training, these changes are correlated with where the agent actually visits, keeping the system stable. Under off-policy training, updates target states that the behavior policy visits frequently but the greedy policy does not — the mismatch destabilizes the weight dynamics.

### Condition 2: Bootstrapping

Using the current value estimate as part of the update target. All TD methods bootstrap:

$$\text{TD target: } \quad R_{t+1} + \gamma \hat{v}(S_{t+1}, \mathbf{w})$$

Monte Carlo methods do not bootstrap — the return $G_t = \sum_{k=0}^\infty \gamma^k R_{t+k+1}$ is computed from actual rewards, not estimates.

**Why bootstrapping matters:** The target depends on $\mathbf{w}$. If $\hat{v}(S_{t+1}, \mathbf{w})$ is an overestimate, the target is inflated, which increases the prediction for $S_t$, which then inflates targets for states that lead to $S_t$, and so on. This feedback loop is bounded under tabular methods and on-policy FA, but can diverge under off-policy FA.

### Condition 3: Off-Policy Training

Training on transitions generated by a behavior policy $b(a|s)$ that differs from the target policy $\pi(a|s)$. Q-learning is the canonical example: it learns the greedy value function while following an $\epsilon$-greedy behavior policy. Experience replay (used in DQN) also creates off-policy data because the replay buffer contains transitions from older policies.

**Why off-policy matters:** The state distribution $\mu(s)$ under the behavior policy differs from the on-policy distribution $d^\pi(s)$. The convergence proof for linear TD requires the update to be weighted by $d^\pi(s)$. When samples come from a different distribution, the matrix $\mathbf{A} = \mathbb{E}_{d^b}[\mathbf{x}(S)(\mathbf{x}(S) - \gamma \mathbf{x}(S'))^T]$ can be non-positive-definite, eliminating the fixed point.

## 2. Why Each Pairwise Combination Is Safe

### Tabular + Bootstrapping + Off-Policy (Safe)

<div class="callout-insight">

<strong>Insight:</strong> ### Tabular + Bootstrapping + Off-Policy (Safe)

Tabular Q-learning converges under off-policy training because each state has its own dedicated parameter — there is no cross-state interference from w...

</div>


Tabular Q-learning converges under off-policy training because each state has its own dedicated parameter — there is no cross-state interference from weight sharing. The off-policy mismatch affects data efficiency (some states are visited less than others) but not stability (each parameter moves independently toward its correct value).

**Convergence:** Watkins & Dayan (1992) prove tabular Q-learning converges with probability 1 given sufficient exploration and appropriate step sizes.

### FA + Bootstrapping + On-Policy (Safe)

On-policy linear TD(0) converges to the TD fixed point (Tsitsiklis & Van Roy, 1997). The on-policy distribution $d^\pi(s)$ ensures that the updates are consistent with where the agent visits — there is no mismatch. The matrix $\mathbf{A}$ is positive definite under on-policy sampling.

### FA + Monte Carlo + Off-Policy (Safe)

Monte Carlo methods with importance sampling correct for the off-policy distribution by reweighting each return by $\rho_t = \pi(A_t|S_t) / b(A_t|S_t)$. The non-bootstrapped target $G_t$ does not depend on $\mathbf{w}$, so the feedback loop that causes divergence does not arise. The update is a true SGD step on VE with a corrected target.

## 3. Baird's Counterexample (Conceptual)

Baird (1995) constructed a minimal example of divergence under the deadly triad: a 7-state MDP with linear function approximation trained off-policy with semi-gradient TD.

### The Setup

```
States: s1, s2, s3, s4, s5, s6 (non-terminal), s7 (special)
Actions: "dashed" (always goes to s7) or "solid" (uniform to s1-s6)
Behavior policy: 6/7 probability solid, 1/7 probability dashed
Target policy: always dashed (always goes to s7)
Reward: 0 everywhere
Discount: gamma = 0.99
```

### Feature Vectors (Linear Approximation)

$$\mathbf{x}(s_1) = [2,0,0,0,0,0,0,1]^T, \quad \mathbf{x}(s_i) = [0,\ldots,0,2,0,1]^T \text{ (2 in position } i\text{)}$$
$$\mathbf{x}(s_7) = [0,0,0,0,0,0,0,2]^T$$

The approximation can represent the true value function exactly: $V^\pi = 0$ everywhere (all rewards are 0), achieved with $\mathbf{w} = \mathbf{0}$.

### What Happens

Despite the true optimum being $\mathbf{w} = \mathbf{0}$ (a zero vector), semi-gradient Q-learning diverges. The weight vector grows without bound.

```
Step    w1      w2      w3      ...   w8
0      1.0    1.0    1.0          1.0
100    1.2    1.2    1.2          1.2
500    3.1    3.1    3.1          3.1
1000  14.2   14.2   14.2         14.2   ← diverging
5000  ∞      ∞      ∞            ∞
```

The divergence occurs because the off-policy mismatch causes updates to push weights in directions that increase all value estimates simultaneously. The bootstrapped target $\gamma \hat{v}(S', \mathbf{w})$ amplifies the existing overestimates, creating a runaway feedback loop.

### Why the True Optimum Is Not Found

Gradient descent would descend to $\mathbf{w} = \mathbf{0}$. But semi-gradient TD does not follow the gradient of any objective — its update direction depends on the off-policy distribution, which creates a net positive push on the weights at $\mathbf{w} = \mathbf{0}$.

```python
# Conceptual illustration of Baird's counterexample
import numpy as np

# Semi-gradient TD update direction at w=0 for Baird's MDP
# is POSITIVE (moves away from zero, not toward it)
def baird_update_direction(w, gamma=0.99):
    """
    Simulate the expected weight update for Baird's 7-state MDP.

    The off-policy distribution causes updates that push w > 0
    even though the true value function is w = 0.
    """
    # Simplified: behavior policy visits s1-s6 with prob 6/7
    # target policy goes to s7 always
    # reward = 0, so delta = gamma * v(s7) - v(si)
    v_s7 = 2 * w[-1]                   # v(s7) = 2 * w8
    v_si = 2 * w[:-1] + w[-1]          # v(si) = 2*wi + w8

    # Expected update (off-policy weighted)
    # Under behavior policy, we mostly visit s1-s6
    delta_si = gamma * v_s7 - v_si     # shape: (6,)
    expected_delta = np.mean(delta_si)

    return expected_delta   # Positive at w=0: moves away from true solution

# At w=0: expected update direction
w_zero = np.zeros(8)
direction = baird_update_direction(w_zero)
print(f"Expected update at w=0: {direction:.4f}")  # Positive → divergence
```

## 4. When Each Combination Fails

```
         Tabular    Linear FA    Nonlinear FA
         -------    ---------    ------------
On-policy   ✓           ✓            ~*
Off-policy  ✓           ✗            ✗

✓  = convergence guaranteed
~* = no guarantee, often works in practice
✗  = can diverge (deadly triad)
```

The tabular row is always safe. The off-policy + FA cells are the danger zone.

## 5. Mitigation Strategies

The deadly triad cannot be avoided in its exact form — any algorithm that uses FA, bootstrapping, and off-policy data has no convergence guarantee. However, several strategies make divergence rare in practice.

### 5.1 Experience Replay (Partially Mitigates)

Store past transitions $(S_t, A_t, R_{t+1}, S_{t+1})$ in a replay buffer. Sample uniformly to train the value function.

**How it helps:** Breaks temporal correlation in the data stream. Correlated sequential data causes the weight vector to track recent biases rather than converging to a stable value.

**How it fails:** The replay buffer still contains off-policy data (transitions from old policies). The deadly triad is still present.

The following implementation builds on the approach above:

```python
from collections import deque
import random

class ReplayBuffer:
    """Fixed-capacity replay buffer with uniform sampling."""

    def __init__(self, capacity=10_000):
        self.buffer = deque(maxlen=capacity)

    def push(self, state, action, reward, next_state, done):
        self.buffer.append((state, action, reward, next_state, done))

    def sample(self, batch_size):
        batch = random.sample(self.buffer, batch_size)
        states, actions, rewards, next_states, dones = zip(*batch)
        return (np.array(states), np.array(actions),
                np.array(rewards), np.array(next_states),
                np.array(dones, dtype=float))

    def __len__(self):
        return len(self.buffer)
```

</div>

### 5.2 Target Networks (Partially Mitigates)

Maintain two networks: the online network $\hat{q}(s, a, \mathbf{w})$ (updated every step) and a target network $\hat{q}(s, a, \mathbf{w}^-)$ (updated infrequently). Use the target network to compute TD targets:

$$\delta_t = R_{t+1} + \gamma \max_{a'} \hat{q}(S_{t+1}, a', \mathbf{w}^-) - \hat{q}(S_t, A_t, \mathbf{w})$$

**How it helps:** The target network provides a slowly-moving bootstrap target. The online network chases a target that does not immediately respond to its own updates, breaking part of the feedback loop.

**How it fails:** The target network is still a function of the same class. The triad is not eliminated, just slowed. On very long training runs, divergence can still occur.

```python
def update_target_network_hard(online_params, target_params):
    """Copy online weights to target (hard update)."""
    target_params.load_state_dict(online_params.state_dict())

def update_target_network_soft(online_params, target_params, tau=0.005):
    """Polyak averaging: target = tau*online + (1-tau)*target."""
    for param, target_param in zip(online_params.parameters(),
                                    target_params.parameters()):
        target_param.data.copy_(
            tau * param.data + (1 - tau) * target_param.data
        )
```

### 5.3 Gradient Correction Methods (Theoretically Sound)

**GTD (Gradient TD)** and **GTD2** methods modify the update rule to follow the true gradient of the projected Bellman error. They add a correction term that accounts for the off-policy mismatch:

$$\mathbf{w} \leftarrow \mathbf{w} + \alpha \left[ \delta_t \mathbf{x}(S_t) - \gamma \mathbf{x}(S_{t+1}) (\mathbf{x}(S_t)^T \mathbf{h}) \right]$$

where $\mathbf{h}$ is a secondary weight vector updated concurrently. GTD methods converge off-policy with linear FA, at the cost of slower convergence and greater complexity.

**Emphatic TD** uses importance sampling weights to correct the state distribution, restoring convergence at the cost of higher variance.

These methods are provably correct but rarely used in practice because DQN-style tricks are simpler and work well empirically.

### 5.4 Reducing Off-Policy-ness

Use on-policy methods (SARSA, A2C, PPO) instead of off-policy methods (Q-learning, DQN) when possible. On-policy methods collect transitions from the current policy, eliminating the off-policy component of the triad.

**Trade-off:** On-policy methods cannot reuse old data (no replay buffer), requiring more environment interactions per gradient step. Off-policy methods are more data-efficient but less stable.

### 5.5 Reducing Bootstrapping

Use $n$-step returns or Monte Carlo returns to reduce the dependency on the bootstrap estimate:

$$G_{t:t+n} = R_{t+1} + \gamma R_{t+2} + \ldots + \gamma^{n-1} R_{t+n} + \gamma^n \hat{v}(S_{t+n}, \mathbf{w})$$

As $n \to \infty$, this approaches the full Monte Carlo return. Larger $n$ reduces the impact of errors in $\hat{v}$ at the cost of higher variance and slower credit assignment.

## 6. Semi-Gradient Methods: Convergence Under On-Policy + Linear FA

Semi-gradient TD(0) and SARSA converge under on-policy + linear FA because:

1. The on-policy distribution $d^\pi(s)$ makes the expected update matrix $\mathbf{A}$ positive definite.
2. Positive definite $\mathbf{A}$ means the fixed point $\mathbf{w}_{TD} = \mathbf{A}^{-1}\mathbf{b}$ exists and is unique.
3. Semi-gradient TD is a linear stochastic approximation (the expected update is $\mathbf{A}\mathbf{w} - \mathbf{b}$), and linear SA with a positive definite $\mathbf{A}$ converges.

Remove on-policy (go off-policy): $\mathbf{A}$ may not be positive definite.
Replace linear FA with nonlinear: The system is no longer linear SA — convergence analysis does not apply.

## 7. Connection to DQN's Innovations

DQN (Mnih et al., 2015) successfully combined all three deadly triad components by mitigating their interaction:

| DQN Innovation | Deadly Triad Ingredient Addressed |
|---|---|
| Convolutional neural network | Function approximation (necessary for Atari) |
| Experience replay buffer | Off-policy training (decorrelates data) |
| Target network (updated every 10k steps) | Bootstrapping (stabilizes target) |
| Gradient clipping | Bootstrapping (prevents runaway updates) |
| Reward clipping to $[-1, 1]$ | Bootstrapping (bounds target magnitude) |

None of these innovations removes a triad ingredient — they reduce the severity of the interaction. DQN still has all three ingredients. It works empirically on many games and fails on others where the triad interactions are severe.

## Common Pitfalls

<div class="callout-danger">

<strong>Danger:</strong> The pitfalls below are the most common mistakes practitioners make. Each one can silently degrade your results without obvious errors.

</div>

**Pitfall 1: Assuming Q-learning + linear FA is safe because Q-learning is safe tabularly.**
Tabular Q-learning converges; Q-learning + linear FA does not. The convergence proof for tabular Q-learning relies on independent updates per state, which FA violates. Baird's counterexample disproves safety for the FA case.

<div class="callout-warning">

<strong>Warning:</strong> **Pitfall 1: Assuming Q-learning + linear FA is safe because Q-learning is safe tabularly.**
Tabular Q-learning converges; Q-learning + linear FA does not.

</div>

**Pitfall 2: Thinking experience replay solves the deadly triad.**
Replay buffers help by decorrelating data but do not remove the off-policy component. They change the effective distribution of training data, making divergence less likely but not impossible. DQN still diverges on some Atari games.

**Pitfall 3: Expecting GTD/GTD2 to be faster than DQN-style methods.**
GTD methods are convergent but converge slowly in practice. The secondary weight vector adds computational overhead and the convergence rate is sublinear. DQN-style heuristics converge faster empirically at the cost of theoretical guarantees.

**Pitfall 4: Using a target network update period that is too short or too long.**
Too short (update every step = hard target): the target moves with the online network, restoring the feedback loop. Too long (update every 100k steps): the target is stale, and the online network may diverge before the target catches up. The DQN paper uses 10k steps; modern methods use soft updates ($\tau = 0.005$).

**Pitfall 5: Using rewards with large variance or magnitude with bootstrapping.**
Large rewards amplify the TD error, which amplifies the weight update. Combined with FA and off-policy training, this accelerates divergence. Reward normalization or clipping is a practical necessity, not an optional improvement.
