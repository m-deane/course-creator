genetic_selector.py
import numpy as np from sklearn.base import BaseEstimator class GeneticSelector(BaseEstimator): """Feature selection using genetic algorithms.""" def __init__(self, estimator, n_features=10, population_size=50, n_generations=100): self.estimator = estimator self.n_features = n_features self.population_size = population_size self.n_generations = n_generations def fit(self, X, y): # Initialize random population self.population_ = self._init_population(X.shape[1]) # Evolve for n_generations for gen in range(self.n_generations): fitness = self._evaluate(X, y) self.population_ = self._evolve(fitness) return self
Wrapper method uses model accuracy as fitness

Callout Boxes

Five semantic callout types, each with a distinct pastel background and accent border.

Insight: Genetic algorithms are particularly effective when the search space is too large for exhaustive methods but has enough structure for evolutionary pressure to exploit. They trade guaranteed optimality for practical scalability.
Warning: Premature convergence occurs when the population loses diversity too early. Monitor population entropy across generations and consider increasing mutation rate if diversity drops below threshold.
Key Point: Tournament selection balances exploitation and exploration better than roulette wheel for most feature selection tasks. Start with tournament size k=3 and adjust based on convergence speed.
Danger: Never evaluate fitness on the test set. Use cross-validation within the training fold only. Data leakage from fitness evaluation is the most common mistake in GA-based feature selection.
Info: The chromosome length equals the number of input features. Each gene is a binary flag (0 or 1) indicating whether that feature is included in the selected subset.

After: GA Feature Selection

  • Captures interaction effects automatically
